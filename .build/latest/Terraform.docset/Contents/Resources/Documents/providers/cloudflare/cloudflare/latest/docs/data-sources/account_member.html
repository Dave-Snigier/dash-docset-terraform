cloudflare_account_member (Data Source)

Example Usage

data "cloudflare_account_member" "example_account_member" {
  account_id = "eb78d65290b24279ba6f44721b3ea3c4"
  member_id = "4536bcfad5faccb111b47003c79917fa"
}

Schema

Required

  • account_id (String) Account identifier tag.

Optional

  • filter (Attributes) (see below for nested schema)
  • member_id (String) Membership identifier tag.

Read-Only

  • id (String) Membership identifier tag.
  • policies (Attributes List) Access policy for the membership (see below for nested schema)
  • roles (Attributes List) Roles assigned to this Member. (see below for nested schema)
  • status (String) A member's status in the account.
  • user (Attributes) Details of the user associated to the membership. (see below for nested schema)

Nested Schema for filter

Optional:

  • direction (String) Direction to order results.
  • order (String) Field to order results by.
  • status (String) A member's status in the account.

Nested Schema for policies

Read-Only:

  • access (String) Allow or deny operations against the resources.
  • id (String) Policy identifier.
  • permission_groups (Attributes List) A set of permission groups that are specified to the policy. (see below for nested schema)
  • resource_groups (Attributes List) A list of resource groups that the policy applies to. (see below for nested schema)

Nested Schema for policies.permission_groups

Read-Only:

  • id (String) Identifier of the group.
  • meta (Attributes) Attributes associated to the permission group. (see below for nested schema)
  • name (String) Name of the group.

Nested Schema for policies.permission_groups.meta

Read-Only:

  • key (String)
  • value (String)

Nested Schema for policies.resource_groups

Read-Only:

  • id (String) Identifier of the group.
  • meta (Attributes) Attributes associated to the resource group. (see below for nested schema)
  • name (String) Name of the resource group.
  • scope (Attributes List) The scope associated to the resource group (see below for nested schema)

Nested Schema for policies.resource_groups.meta

Read-Only:

  • key (String)
  • value (String)

Nested Schema for policies.resource_groups.scope

Read-Only:

  • key (String) This is a combination of pre-defined resource name and identifier (like Account ID etc.)
  • objects (Attributes List) A list of scope objects for additional context. (see below for nested schema)

Nested Schema for policies.resource_groups.scope.objects

Read-Only:

  • key (String) This is a combination of pre-defined resource name and identifier (like Zone ID etc.)
