Data Source: aws_ssm_instances

Use this data source to get the instance IDs of SSM managed instances.

Example Usage

data "aws_ssm_instances" "example" {
  filter {
    name   = "PlatformTypes"
    values = ["Linux"]
  }
}

Argument Reference

  • filter - (Optional) Configuration block(s) for filtering. Detailed below.

filter Configuration Block

The filter configuration block supports the following arguments:

  • name - (Required) Name of the filter field. Valid values can be found in the SSM InstanceInformationStringFilter API Reference.
  • values - (Required) Set of values that are accepted for the given filter field. Results will be selected if any given value matches.

Attribute Reference

This data source exports the following attributes in addition to the arguments above:

  • ids - Set of instance IDs of the matched SSM managed instances.
