Resource Type definition for Metric Stream

Example Usage

Stream CloudWatch Metrics to S3 via Firehose

Sets up a CloudWatch Metric Stream that captures specific EC2 metrics (CPUUtilization, NetworkIn, NetworkOut) and forwards them through Kinesis Firehose to an S3 bucket, with all necessary IAM roles and permissions configured.

Schema

Optional

  • exclude_filters (Attributes List) Define which metrics will be not streamed. Metrics matched by multiple instances of MetricStreamFilter are joined with an OR operation by default. If both IncludeFilters and ExcludeFilters are omitted, all metrics in the account will be streamed. IncludeFilters and ExcludeFilters are mutually exclusive. Default to null. (see below for nested schema)
  • firehose_arn (String) The ARN of the Kinesis Firehose where to stream the data.
  • include_filters (Attributes List) Define which metrics will be streamed. Metrics matched by multiple instances of MetricStreamFilter are joined with an OR operation by default. If both IncludeFilters and ExcludeFilters are omitted, all metrics in the account will be streamed. IncludeFilters and ExcludeFilters are mutually exclusive. Default to null. (see below for nested schema)
  • include_linked_accounts_metrics (Boolean) If you are creating a metric stream in a monitoring account, specify true to include metrics from source accounts that are linked to this monitoring account, in the metric stream. The default is false.
  • name (String) Name of the metric stream.
  • output_format (String) The output format of the data streamed to the Kinesis Firehose.
  • role_arn (String) The ARN of the role that provides access to the Kinesis Firehose.
  • statistics_configurations (Attributes List) By default, a metric stream always sends the MAX, MIN, SUM, and SAMPLECOUNT statistics for each metric that is streamed. You can use this parameter to have the metric stream also send additional statistics in the stream. This array can have up to 100 members. (see below for nested schema)
  • tags (Attributes List) A set of tags to assign to the delivery stream. (see below for nested schema)
